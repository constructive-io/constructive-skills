# Identity Columns

Native PostgreSQL identity columns â€” `GENERATED ALWAYS AS IDENTITY` and
`GENERATED BY DEFAULT AS IDENTITY` â€” with optional sequence tuning
(`START` / `INCREMENT` / `MINVALUE` / `MAXVALUE` / `CACHE` / `CYCLE`).

An identity column is backed by an implicit sequence the database manages for
you. Prefer identity over a manual `bigserial` + default: it is the modern
SQL-standard form, keeps the owning sequence tied to the column's lifecycle, and
(with `ALWAYS`) prevents clients from supplying their own value.

Configured on a field through two ORM columns:

| ORM field         | Type                | Meaning |
|-------------------|---------------------|---------|
| `identityGeneration` | `'always' \| 'by_default'` | Turns the field into an identity column. Omit (leave `null`) for a normal field. |
| `identityOptions`    | object (optional)   | Sequence tuning for the backing identity sequence. |

`type` should be an integer type â€” `bigint` (recommended), `integer`, or
`smallint`.

---

## `GENERATED ALWAYS AS IDENTITY`

The database always assigns the value; clients cannot write the column (inserts
that supply it are rejected unless they use `OVERRIDING SYSTEM VALUE`).

```typescript
await db.field.create({
  data: {
    databaseId,
    tableId,
    name: 'id',
    type: { name: 'bigint' },
    identityGeneration: 'always',
  },
  select: { id: true },
}).execute();
```

Generates:

```sql
id bigint GENERATED ALWAYS AS IDENTITY
```

---

## `GENERATED BY DEFAULT AS IDENTITY`

The database assigns a value when the client omits the column, but a client may
also supply an explicit value (useful for data imports / replication).

```typescript
await db.field.create({
  data: {
    databaseId,
    tableId,
    name: 'n',
    type: { name: 'bigint' },
    identityGeneration: 'by_default',
    identityOptions: { start: 100, increment: 5 },
  },
  select: { id: true },
}).execute();
```

Generates:

```sql
n bigint GENERATED BY DEFAULT AS IDENTITY (START WITH 100 INCREMENT BY 5)
```

## Sequence options (`identityOptions`)

All keys are optional; omit the object entirely to accept PostgreSQL defaults.

```typescript
interface IdentityOptions {
  start?: number;      // START WITH   â€” first value (default 1)
  increment?: number;  // INCREMENT BY â€” step (default 1)
  min?: number | null; // MINVALUE     â€” null / omitted uses the type default
  max?: number | null; // MAXVALUE     â€” null / omitted uses the type default
  cache?: number;      // CACHE        â€” values preallocated per session (default 1)
  cycle?: boolean;     // CYCLE / NO CYCLE â€” wrap at bounds (default false â†’ NO CYCLE)
}
```

Only keys with a meaningful value are emitted. A `null` (or omitted) `min`/`max`
falls back to the sequence's type default, and `cycle: false` (or omitted) emits
nothing (the column is `NO CYCLE`).

| `identityOptions` | Emitted clause |
|-------------------|----------------|
| `undefined` / `{}` | *(none)* |
| `{ start: 100, increment: 5 }` | `(START WITH 100 INCREMENT BY 5)` |
| `{ start: 10, increment: 2, min: 1, max: 999, cache: 4, cycle: true }` | `(START WITH 10 INCREMENT BY 2 MINVALUE 1 MAXVALUE 999 CACHE 4 CYCLE)` |
| `{ start: 100, increment: 5, min: null, max: null, cache: 1, cycle: false }` | `(START WITH 100 INCREMENT BY 5 CACHE 1)` |

Full example:

```typescript
await db.field.create({
  data: {
    databaseId,
    tableId,
    name: 'ticket_no',
    type: { name: 'bigint' },
    identityGeneration: 'always',
    identityOptions: { start: 10, increment: 2, min: 1, max: 999, cache: 4, cycle: true },
  },
  select: { id: true },
}).execute();
```

Generates:

```sql
ticket_no bigint GENERATED ALWAYS AS IDENTITY (START WITH 10 INCREMENT BY 2 MINVALUE 1 MAXVALUE 999 CACHE 4 CYCLE)
```

## Rules & behavior

- **Mutually exclusive.** A field may set at most one of `identityGeneration`,
  `generationExpression` (generated column), or `defaultValue`. Combining them is
  rejected:
  - identity + `generationExpression` â†’ `identity column cannot also be a generated column`
  - identity + `defaultValue` â†’ `identity column cannot have a default value`
- **Allowed values.** `identityGeneration` must be `'always'` or `'by_default'`;
  any other value is rejected (`identity_generation must be always or by_default`).
- **Immutable generation mode.** `identityGeneration` cannot be changed after the
  field is created (Phase 1). Create the field with the mode you want.
- **Hidden from create input.** Identity fields are automatically tagged
  `{"behavior": "-insert"}`, so the generated GraphQL create input omits them â€”
  clients never provide the value on insert. (`ALWAYS` columns additionally
  reject any client-supplied value at the database level; `BY DEFAULT` columns
  can still be written directly via SQL / imports.)
- **Implicitly NOT NULL.** Identity columns are `NOT NULL` by definition; you do
  not need `isRequired: true`.

## Verifying

```typescript
const field = await db.field.findOne({
  where: { id: fieldId },
  select: { id: true, name: true, identityGeneration: true, identityOptions: true },
}).execute();
```

At the physical level the column reports `pg_attribute.attidentity = 'a'` for
`always` and `'d'` for `by_default`, and the backing sequence's `start_value` /
`increment_by` reflect `identityOptions`.
